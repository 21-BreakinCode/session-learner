# Session Take-Away

Analyze the current conversation and produce a reflective summary. Do NOT save to any file — display in the terminal only.

## Instructions

Review the full conversation history in this session. Produce the following sections:

### 1. Session Reflection

Write a conversational, honest reflection covering:

- **Misconceptions corrected**: Where did the user misunderstand something? What was the correction? Why does it matter?
- **Confusions clarified**: What concepts or decisions were unclear at first? How were they resolved?
- **What went well**: What practices, decisions, or approaches worked effectively?
- **Key technical insights**: Any non-obvious technical details worth remembering.

Keep this natural and specific — not generic advice. Reference actual topics from the session.

### 2. Zettelkasten Card Suggestions

Suggest 2-5 atomic knowledge cards the user could create in their Zettelkasten. For each card:

- **Title**: A concise, searchable title (e.g., "Bash Hook Exit Codes in Claude Code")
- **Content outline**: 1-2 sentences describing what the card should capture
- **Suggested links**: 2-3 related topic tags or card titles this could link to

Focus on knowledge that is:
- Reusable across projects (not one-time fixes)
- Conceptual or pattern-based (not implementation-specific details)
- Worth retrieving months from now

### 3. Format

Output everything directly in the terminal. Use markdown formatting. No file creation.
